# SOLID — checklist consolidado

## SRP

- [ ] Uma razão para mudar por módulo
- [ ] God class split

## OCP

- [ ] Extensão via new handler/class
- [ ] Switch estável ou registry

## LSP

- [ ] Subtypes substituíveis
- [ ] Sem throw surprise

## ISP

- [ ] Interfaces pequenas
- [ ] Sem NotImplemented stubs

## DIP

- [ ] Domain depende de abstração
- [ ] Infra implementa port
- [ ] Constructor injection

## Anti-padrões

- SOLID theater (interfaces vazias)
- Over-split classes
- DIP everywhere
