# UX — modelo mental raiz

## O que importa

UX é **qualidade da experiência ao completar tarefas** — não só estética. Mede-se por: tempo até objetivo, erros, retrabalho, confiança. Decisões baseadas em fluxo do usuário, não preferência interna.

## Job to be done

"Pessoa X precisa Y no contexto Z" — toda tela responde a isso.

## Carga cognitiva

Reduza decisões por tela; defaults inteligentes; progressive disclosure (mostre avançado só quando pedido).

## Consistência

Padrões repetidos (navegação, ações primárias, feedback) reduzem aprendizado.

## Fronteiras

| Tópico | Skill |
|---|---|
| Visual/contraste | `ui` |
| HTML/a11y técnico | `html` |
| React implementation | `react` |

## Anti-padrões

- Feature dump sem hierarquia de tarefa.
- Copiar Dribbble sem contexto de uso real.
- Dark patterns (confirm shaming, cancel difícil).
