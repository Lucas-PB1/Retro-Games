# UI — modelo mental raiz

## O que importa

UI é **linguagem visual** que comunica hierarquia, estado e affordance. Consistência de tokens (cor, tipo, espaço) cria sistema previsível — não telas isoladas bonitas.

## Design system mindset

Componentes reutilizáveis + tokens + estados documentados. Customização pontual quebra escala.

## Affordance

Elemento **parece** clicável/editável: botão com contraste, input com border, link sublinhado ou cor distinta.

## Densidade

Dashboard pro ≠ consumer mobile — calibre padding, font-size e informação por contexto.

## Fronteiras

| Tópico | Skill |
|---|---|
| CSS/Tailwind | `css`, `tailwind` |
| UX fluxo | `ux` |
| HTML semântico | `html` |

## Anti-padrões

- 5 cores primárias diferentes por página.
- Gray text ilegível "por estética".
- Ícone sem label onde significado não é universal.
